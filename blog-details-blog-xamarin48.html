Drag and Drop is Here!



You heard it right, drag and drop finally available in Xamarin.Forms 4.8! The new feature allows you to drag objects right into an application, from one location to another or even, drag an object from one app to another. This is an extremely exciting step forward for .NET and Xamarin considering it was time consuming for developers to manually move objects from one location to another, be it a location, application for a new platform. This has made the lives of Xamarin developers much easier.




Credit to Jake Kirsch, Microsoft


Keep in mind, you will have to add a DragGestureRecognizer for anything youâ€™d like to drag and drop:



  • 1. Add a DragGestureRecognizer behavior to the objectâ€™s GestureRecognizers collection found in the toolbar.
  • 2. Build a data package. With Xamarin.Forms 4.8, the program will automatically render and create the data package for you.
  • 3. Enable DragGestureRecognizer.AllowDrop property to True.
  • 4. If necessary, set the DragGestureRecognizer.DragOver to indicate which event should occur when your mouse rolls over the object.
  • 5. Your object is now draggable to any location or application.


Make Gradients with 4.8



Xamarin.Forms has now created prototype brushes and gradients for the experimental crowd of developers. Specifically, the brush tool allows you to select a shape and fill it with a color or gradient. For now, there are three different types of brushes: fill (solid colors), gradient (gradients) and radial (radial gradients). Implementing a gradient is as easy as adding some more code to the snippet above. Configure the StartPoint and EndPoint properties for your gradient and youâ€™re off to the races. Radial gradients are great alternatives to regular gradients, and can also be bindable, like the gradients, to your data. Letâ€™s Put Those Gradients To Use - Menu Flyouts



Time to bring those gradients into live interfaces. A creative and effective way to use the gradient XAML (and is now supported with 4.8) is to use it as a backdrop for mobile view menu selection. The cool new feature with Xamarin.Forms is the ability to fade that gradient in as the end user taps that hamburger icon and opens the mobile view menu flyout. Shown above is after the user has the menu selected. Can you spot the change? The gradient has appeared behind the menu. It can be a useful feature for stylized web apps. Letâ€™s take a look at how we can implement this into our projects:

  • 1. Set the Shell.FlyoutBackdrop property to a Brush.

  • 2. Define if youâ€™d like a Solid fill, Radial or traditional Gradient.

  • 3. Assign the brush to the Shell.FlyoutBackdrop property.



Here is what the XAML should look like for the shell:




<Shell >
        <FlyoutItem >
          <Shell.FlyoutBackdrop>
              <LinearGradientBrush StartPoint="0,0"
                                   EndPoint="1,1">
                  <GradientStop Color="#8A2387"
                                Offset="0.1" />
                  <GradientStop Color="#E94057"
                                Offset="0.6" />
                  <GradientStop Color="#F27121"
                                Offset="1.0" />
              </LinearGradientBrush>
          </Shell.FlyoutBackdrop>
        </FlyoutItem>
</Shell>


And there you have it. A gradient or fill color that shows up when the user selects the mobile menu selection, very useful and unique from most other application menus and very useful for many installations!
